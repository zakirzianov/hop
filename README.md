# 🌊 Hop Traffic Manager

**Скрипт для переадресации трафика (NAT) и ускорения сети на Linux.**

Cоздание "мостов" к VPN (AmneziaWG, WireGuard) и Proxy (VLESS, XRay).


---

## 🌟 Возможности

* **🚀 Быстрый запуск:** После первой установки скрипт доступен из любой папки по команде `hop`.
* **⚡ High Speed Core:** Работает через **Iptables (Kernel NAT)**. Никаких лишних процессов, скорость ограничена только каналом сервера.
* **🏎 BBR Turbo:** Автоматически включает алгоритм **Google BBR** для максимального ускорения TCP соединений.
* **🔀 Мультипротокольность:**
    * Поддержка **UDP** (AmneziaWG, WireGuard).
    * Поддержка **TCP** (VLESS, VMess, Reality).
* **🛠 Кастомные правила:** Возможность указывать разные порты для входа и выхода (проброс SSH, RDP и нестандартных сервисов).
* **🌐 Мульти-туннелирование:** Создавайте 2, 5, 10 соединений на разных портах одновременно.
* **🧠 Умная настройка:**
    * Автоматическое открытие портов в UFW.
    * Умное удаление правил `FORWARD` (файрвол остается чистым даже при разных портах).
    * Сохранение правил после перезагрузки (`netfilter-persistent`).
* **📱 Удобное меню:** Просмотр списка правил, удаление по одному, полный сброс настроек.

## 🚀 Быстрая установка

Подключитесь к вашему VPS (Ubuntu/Debian) под пользователем `root` и выполните одну команду:

```bash
wget -O install.sh https://raw.githubusercontent.com/zakirzianov/hop/main/install.sh && chmod +x install.sh && ./install.sh
```

### ⚡ Последующие запуски
При первой установке скрипт автоматически прописывается в систему. В дальнейшем, чтобы открыть меню, просто введите в терминале:
```bash
hop
```

## 📚 Инструкция: Как настроить Hop (проброс трафика)

Hop позволяет вам подключаться к локальному серверу, который будет незаметно пересылать весь трафик на ваш конечный сервер.

### Шаг 1: Подготовка
У вас должны быть на руках данные от вашего **конечного** сервера:
* **IP-адрес** (например: `40.198.25.32`)
* **Порт** (на котором работает целевой сервис, например 51820 или 443)

### Шаг 2: Настройка этого сервера
1. Запустите скрипт командой `hop`.
2. В меню выберите нужный протокол или "кастомное правило".
3. Введите IP-адрес конечного сервера и порт(ы).
4. Скрипт сообщит об успешном создании "моста".

### Шаг 3: Настройка клиента (Важно!)
1. Откройте файла конфигурации конечного vpn сервера (WG/AWG или Vless ).
2. В настройках соединения найдите поле **Endpoint / Адрес сервера**.
3. Замените зарубежный IP на **IP-адрес этого сервера**.
4. Если вы использовали разные порты в п.3 скрипта (кастом), укажите Входящий порт.
5. Сохраните настройки и подключитесь. Готово! Теперь трафик идет: Клиент -> Hop Сервер -> Конечный сервер!

### Пояснения:
Сервер Hop играет роль промежуточного узла (relay / forwarder), через который проходит трафик.
Принимает входящий трафик и перенаправляет его дальше (iptables DNAT). 

**Что видит интернет-провайдер (в РФ у клиента)**

Провайдер видит только:
- IP твоего RU VPS (Hop)
- факт постоянного соединения (UDP/TCP)
- объём трафика
- время активности

❗ Он НЕ видит:
- EU VPS и его IP
- сайты
- содержимое трафика
👉 Для провайдера ты просто “подключился к серверу в РФ”


**Что видит интернет (сайты / сервисы)**

Сайты видят:
- IP EU VPS

❗Они НЕ видят:
- твой реальный IP
- RU VPS
- цепочку
